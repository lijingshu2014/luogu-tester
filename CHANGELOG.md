# Change Log

“luogu-tester” 插件的所有重大变更均会记录于此文件。

## [1.1.2] - 2025-11-21

### 新增

- 无

### 更改

- 更改为启动 vscode 自动运行插件。

### 移除

- 移除需要依赖工作区中含有 cpp 文件才能启动插件的逻辑。

## [1.1.1] - 2025-11-16

### 新增

- 新增根据代码文件所在目录生成可执行文件的逻辑。
- 新增根据操作系统动态命名可执行文件的逻辑（Windows 为 “文件名.exe”，Linux/Mac 为 “文件名”）。
- 新增动态获取代码文件所在目录、代码文件名（不含后缀）的工具逻辑。

### 更改

- 更改可执行文件生成路径：从工作区根目录改为代码文件所在目录。
- 更改可执行文件清理提示信息：从固定提示 “a.out” 改为动态显示实际生成的可执行文件名。
- 更改可执行文件路径计算逻辑：从 “工作区根目录 + a.out” 改为 “代码所在目录 + 系统对应可执行文件名”。

### 移除

- 移除固定生成 “a.out” 可执行文件的命名逻辑。
- 移除固定使用工作区根目录作为可执行文件生成目录的逻辑。

## [1.1.0] - 2025-11-08

### 新增
- 完善评测结果分类，新增 CE（编译错误）、TLE（时间超限）、RE（运行时错误）三种明确结果。
- 编译错误时显示详细错误信息（前500字符）。
- 运行时错误区分信号类型（段错误、异常终止等）并给出中文说明。
- 输出对比时忽略行尾空格和空行差异，更符合实际评测场景。
- 评测完成后自动清理生成的可执行文件（a.out）。
- AC 结果新增执行时间和内存使用统计展示。

### 更改
- 优化 WA（答案错误）结果展示格式，清晰区分“期望输出”与“实际输出”。
- 优化 TLE（时间超限）判定逻辑，同时捕获主动超时信号和运行耗时超限两种场景。
- 编译阶段输出从“继承控制台”改为“捕获后按需展示”，避免控制台冗余输出。
- 统一各类评测结果的提示文案风格，增加对应图标标识（✅/❌/⏱️/💥）。

### 移除
- 移除原有的“简单二元结果（正确/错误）”展示逻辑。
- 移除编译阶段无意义的控制台输出打印。
- 移除评测完成后残留的可执行文件（a.out）保留逻辑。
